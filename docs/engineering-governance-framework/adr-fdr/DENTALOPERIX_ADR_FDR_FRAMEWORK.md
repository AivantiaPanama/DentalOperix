# ADR / FDR Framework

**Generated:** 2026-06-25  
**Framework:** DentalOperix Engineering & Governance Framework  
**Status:** Reusable framework component.

## Purpose

Reusable decision-record structure for architecture decisions and functional decisions.

## ADR Required Fields

- Context.
- Decision.
- Alternatives considered.
- Consequences.
- Affected domains.
- Governance status.

## FDR Required Fields

- Functional behavior.
- Accepted/rejected alternatives.
- Domain ownership.
- Traceability references.

## Non-Reopening Rule

Accepted decisions remain closed unless new formal documentary evidence requires review.
