# 16.3-E — Finance Read Model Pilot Implementation

## Objective

Implement the Finance read-domain pilot as an isolated, read-only domain in the Read Model platform.

## Scope

Implemented Finance read coverage for:

- Invoices
- Payments
- Collections
- FinancialKPIs

Explicitly excluded:

- Reconciliation
- Accounting entries
- Ledger posting

## Architecture

```text
ReadModelSourceProvider
        ↓
FinanceAggregateReadService
        ↓
InvoiceReadAdapter
PaymentReadAdapter
CollectionReadAdapter
FinancialKpiReadAdapter
        ↓
Finance Read Models
        ↓
Financial Projection fallback
```

## Governance

Finance remains isolated from Patient, CRM, Billing, Clinical, and Operations aggregates. It does not introduce cross-domain aggregate calls.

## Observability

ReadModelSourceProvider emits Finance telemetry through ReadObservabilityProvider:

- ReadTelemetryEvent
- AggregateTelemetryEvent
- DomainTelemetryEvent
- FallbackTelemetryEvent

## Fallback

Finance follows the platform fallback policy:

```text
Read Model
↓
Financial Projection
↓
Error
```

The current implementation returns an empty Finance aggregate when falling back through the legacy source path, preserving public contracts and avoiding synthetic financial state.

## Contract safety

No public API contract was changed. Finance telemetry and diagnostics remain internal.
