# 17.3-B — Executive Dashboard UI Architecture Design

## Estado

COMPLETADO — Architecture-only design.

## Objetivo

Definir la arquitectura futura de la capa UI para Executive Dashboard sin implementar UI funcional, rutas nuevas, persistencia, acceso a dominios, acceso a aggregates, acceso a adapters ni exposición de telemetría cruda.

## Resultado

APPROVED FOR COMPONENT DESIGN.

La fase 17.3-B introduce únicamente contratos internos de arquitectura para expresar:

- superficies de dashboard autorizadas;
- consumo exclusivo de ExecutiveDashboardApiContracts;
- separación entre Executive, Operational y Governance Dashboard;
- guardrails de gobernanza UI;
- dependencias prohibidas;
- transición controlada hacia 17.3-C Dashboard Component Design.

- `src/server/read-models/executive-dashboard-ui-architecture.test.ts`

## Guardrails confirmados

- No UI implementation.
- No route implementation.
- No public API expansion.
- No persistence.
- No lead writes.
- No aggregate access.
- No adapter access.
- No raw telemetry exposure.
- Aggregate Isolation preserved.
- Domain Ownership preserved.
- Leads remains Source of Truth.
- READ MODEL PLATFORM v2 remains CLOSED/FROZEN/GOVERNANCE BASELINED.
