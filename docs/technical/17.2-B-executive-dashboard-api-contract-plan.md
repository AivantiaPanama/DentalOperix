# 17.2-B — Executive Dashboard API Contract Plan

## Objective

Define internal API contracts for Executive Observability dashboard metrics without implementing routes, UI, persistence, or new business domains.

## Scope

Included:

- Executive dashboard API contract metadata
- Versioned API response envelopes
- Metric-only payload validation
- Snapshot contract for executive, operational, and governance dashboard contracts

Excluded:

- UI dashboards
- Route implementation
- Raw telemetry exposure
- Functional domain data exposure
- Aggregate or adapter access

## Architecture

The API contracts sit above `executive-dashboard-contracts.ts` and consume only dashboard metric contracts produced from `ExecutiveObservabilityProvider`.

Allowed dependency path:

```text
ReadObservabilityProvider
  -> ExecutiveObservabilityProvider
  -> executive-dashboard-contracts
  -> executive-dashboard-api-contracts
```

Forbidden dependency path:

```text
executive-dashboard-api-contracts
  -> Aggregates / Adapters / ReadModelSourceProvider / UI
```

## API Contract Candidates

- `GET /api/internal/executive-observability/executive`
- `GET /api/internal/executive-observability/operational`
- `GET /api/internal/executive-observability/governance`
- `GET /api/internal/executive-observability/snapshot`

These are contract definitions only. No route files were added in this phase.

## Governance Validation

- Executive Observability remains an infrastructure layer.
- API contracts are metric-only.
- Raw telemetry, diagnostics, resolved identity, and functional domain records are not exposed.
- No UI was introduced.
- No write path was introduced.

## Validation

- API contract tests added.
- Metric-only guard added for API contracts.
- Build and full test suite pass.

## Result

17.2-B establishes the internal API contract plan required before implementing any Executive Dashboard API endpoints.
