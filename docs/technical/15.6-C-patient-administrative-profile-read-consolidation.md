# 15.6-C — Patient Administrative Profile Read Consolidation

## Objetivo

Consolidar la lectura de `PatientAdministrativeProfiles` dentro del agregado de paciente sin modificar contratos públicos, rutas, UI ni escritura.

## Alcance aplicado

- Se agregó `PatientAdministrativeProfileReadAdapter` como capacidad explícita de lectura.
- `PatientAggregateReadService` ahora resuelve internamente el perfil administrativo seleccionado.
- La respuesta pública sigue exponiendo `PatientAdministrativeProfile[]` sin diagnósticos internos.
- Las escrituras administrativas siguen usando Leads como Source of Truth.

## Arquitectura validada

```text
ReadModelSourceProvider
        ↓
PatientAggregateReadService
        ↓
PatientReadAdapter
PatientIdentifierReadAdapter
PatientContactReadAdapter
PatientAdministrativeProfileReadAdapter
        ↓
PatientAdministrativeProfile[] público
```

## Reglas de selección del perfil administrativo

Cuando existen varios perfiles administrativos para un paciente, el adapter prioriza:

1. perfil verificado
2. fecha `updatedAt` más reciente
3. mayor completitud de campos administrativos
4. orden determinista por `profileId`
