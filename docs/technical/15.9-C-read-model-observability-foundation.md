# 15.9-C — Read Model Observability Foundation

## Scope

Implemented the initial observability foundation for the Read Model platform.

## Added

- `src/server/read-models/read-observability-provider.ts`
- `src/server/read-models/read-observability-provider.test.ts`
- Read source telemetry in `ReadModelSourceProvider`
- Fallback telemetry for Patient, CRM, and Billing domains
- Aggregate telemetry for Patient, CRM, and Billing aggregates
- Domain telemetry for Patient, CRM, and Billing domains
- `docs/adr/ADR-018-read-model-observability-foundation.md`
- `docs/technical/read-model-observability-standard.md`

## Architectural constraints preserved

- Leads remains the source of truth.
- No dual write was introduced.
- No productive migration was introduced.
- Public API contracts remain unchanged.
- Domain boundaries remain isolated.
- Telemetry is best effort and cannot fail read requests.

## Instrumentation pattern

```text
ReadModelSourceProvider
        ↓
ReadObservabilityProvider
        ↓
Patient / CRM / Billing aggregate telemetry
```

## Validation focus

- Telemetry event creation
- Best-effort sink behavior
- Read model source usage telemetry
- Fallback telemetry across domains
- Contract isolation
