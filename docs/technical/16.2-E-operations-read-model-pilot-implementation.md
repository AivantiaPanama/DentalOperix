# 16.2-E — Operations Read Model Pilot Implementation

## Status

Implemented and validated.

## Scope

This phase introduces the Operations read domain as an isolated, read-only consumer of worksheet read models.

Included:

- AutomationRuns
- OperationalKPIs
- WorkflowExecutionStatus

Excluded:

- TaskQueues
- ExecutionCommands
- Runtime execution engines

## Architecture

```text
ReadModelSourceProvider
        ↓
OperationsAggregateReadService
        ↓
AutomationRunReadAdapter
OperationalKpiReadAdapter
WorkflowExecutionReadAdapter
        ↓
Operations Read Models
```

## Domain boundary

Operations remains isolated from Patient, CRM, Billing, and Clinical aggregates. The aggregate owns operational read concerns only.

## Source of truth

Leads and existing operational sources remain the source of truth. This phase does not introduce dual write, migration, backfill, or read-model persistence.

## Observability

The ReadModelSourceProvider emits Operations telemetry through ReadObservabilityProvider:

- ReadTelemetryEvent
- FallbackTelemetryEvent
- AggregateTelemetryEvent
- DomainTelemetryEvent

## Fallback

Operations follows the platform fallback policy:

```text
Read Model
    ↓
Operational Projection / Lead Projection
    ↓
Error
```

## Public contracts

No public API contract was changed. Operations telemetry and diagnostics remain internal.
