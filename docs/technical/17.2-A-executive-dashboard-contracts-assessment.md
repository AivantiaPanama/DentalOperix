# 17.2-A — Executive Dashboard Contracts Assessment

## Status

Implemented and validated.

## Objective

Define stable, metric-only contracts for future executive, operational, and governance dashboards without implementing UI and without exposing domain payloads.

## Scope

Included:

- Executive dashboard contract envelope
- Operational dashboard contract envelope
- Governance dashboard contract envelope
- Metric catalog for platform, domain, aggregate, and governance metrics
- Metric-only payload guard
- Contract tests

Excluded:

- Dashboard UI
- New routes
- New persistence
- New read domains
- Direct aggregate or adapter access

## Architecture

The dashboard contract layer consumes only `ExecutiveObservabilityProvider`, which already consumes telemetry events from `ReadObservabilityProvider`.

Allowed source:

```text
ReadObservabilityProvider
↓
ExecutiveObservabilityProvider
↓
Executive Dashboard Contracts
```

Forbidden sources:

```text
PatientAggregateReadService
CRMReadAggregateService
BillingReadAggregateService
ClinicalAggregateReadService
OperationsAggregateReadService
FinanceAggregateReadService
InventoryAggregateReadService
SupportAggregateReadService
Read Adapters
Read Models
UI Components
```

## Contracts

Version:

```text
executive-dashboard-contracts/v1
```

Audiences:

- executive
- operational
- governance

Metric groups:

- PlatformHealthMetric
- DomainHealthMetric
- AggregateHealthMetric
- GovernanceHealthMetric

## Governance validation

Confirmed:

- Executive dashboard contracts are infrastructure, not a new domain.
- Contracts expose metrics only.
- Contracts do not expose telemetry, diagnostics, resolved identity, or functional payloads.
- Contracts do not import aggregates, adapters, read model source provider, or React UI.

## Restrictions

No changes were made to:

- BookingDialog
- processDentalLead
- /api/leads/create
- Calendar
- Gmail
- FloatingDentalAIChat
- Home
- siteServices.ts

## Result

17.2-A establishes a stable contract foundation for future executive dashboards while preserving Platform v2 governance, aggregate isolation, and Source of Truth rules.
