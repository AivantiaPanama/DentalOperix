# 16.3-F — Finance Read Model Pilot Validation & Closure

## Status

Validated.

## Scope Validated

The Finance read domain v1 is limited to read-only reporting entities:

- Invoices
- Payments
- Collections
- FinancialKPIs

The following accounting-oriented capabilities remain explicitly excluded from this pilot:

- Reconciliation
- Accounting Entries
- Ledger Posting

## Architecture Validated

```text
ReadModelSourceProvider
        ↓
FinanceAggregateReadService
        ↓
InvoiceReadAdapter
PaymentReadAdapter
CollectionReadAdapter
FinancialKpiReadAdapter
        ↓
Finance Read Models
        ↓
Fallback → Financial Projection
```

## Domain Ownership

Finance owns only financial reporting read models. It does not own Patient, CRM, Billing, Clinical, or Operations aggregates.

## Aggregate Isolation

The Finance aggregate does not consume other domain aggregate services. Domain association must occur through identifiers and read model projections, not aggregate-to-aggregate coupling.

## Fallback Governance

Finance follows the platform policy:

```text
Read Model
↓
Financial Projection
↓
Error
```

## Observability

Finance emits platform-standard observability through `ReadObservabilityProvider`:

- ReadTelemetryEvent
- FallbackTelemetryEvent
- AggregateTelemetryEvent
- DomainTelemetryEvent

## Public Contracts

No public API contract is modified by this phase. Finance diagnostics and observability remain internal.

## Restrictions Verified

No changes are made to:

- BookingDialog
- processDentalLead
- /api/leads/create
- Calendar
- Gmail
- FloatingDentalAIChat
- Home
- siteServices.ts

## Source of Truth

Leads and upstream operational/financial sources remain the source of truth. The Finance read domain is not a transactional persistence layer and does not introduce dual write, migration, backfill, or lead replacement.

## Closure Result

Finance Read Domain v1 is validated for read-model platform participation with reporting-only scope.
