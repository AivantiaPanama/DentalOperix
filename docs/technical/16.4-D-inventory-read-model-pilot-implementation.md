# 16.4-D — Inventory Read Model Pilot Implementation

## Scope

Implemented the initial Inventory read-domain pilot as an isolated, read-only domain.

Included:

- Products
- Consumables
- StockLevels
- Warehouses

Excluded:

- StockMovements
- InventoryAdjustments
- PurchaseOrders
- SupplierOrders
- Reconciliation

## Architecture

```text
ReadModelSourceProvider
        ↓
InventoryAggregateReadService
        ↓
ProductReadAdapter
ConsumableReadAdapter
StockLevelReadAdapter
WarehouseReadAdapter
        ↓
Inventory Read Models
```

## Governance validation

Inventory remains isolated from:

- Patient
- CRM
- Billing
- Clinical
- Operations
- Finance

The aggregate service does not import other aggregate services.

## Observability

Inventory participates in standard platform observability with domain `Inventory`.

## Source of truth

Inventory read models remain read-only consumers. They do not replace operational inventory systems, Leads, or any future transactional source of truth.
