# 61.0 Module Catalog

Status: ACTIVE
Owner: Architecture / Product Governance
Last updated: 2026-06-20

## Purpose

This catalog defines DentalOperix modules, ownership boundaries, source-of-truth rules, AI assignment candidates, and implementation priority.

## Domain Boundaries

| Domain                   | Source of Truth              | Notes                                                              |
| ------------------------ | ---------------------------- | ------------------------------------------------------------------ |
| Commercial / Acquisition | Leads                        | Certified source for lead capture and appointment origin           |
| Identity / Access        | Users + Roles                | Must not replace Leads or Patients                                 |
| Patient Operations       | Patients                     | Patient identity and clinic relationship, linked to Lead origin    |
| Appointment Operations   | Appointments                 | Operational lifecycle derived from lead booking and clinic actions |
| Analytics                | Read models / views          | Read-only, no write-back                                           |
| Clinical Intelligence    | Derived clinical read models | Read-only unless explicitly approved in future program             |

## Module Implementation Rule

Every module must declare:

- Source of truth.
- Read/write ownership.
- Protected components touched or not touched.
- RBAC boundary.
- Dashboard impact.
- Tests.
- Documentation updates.
