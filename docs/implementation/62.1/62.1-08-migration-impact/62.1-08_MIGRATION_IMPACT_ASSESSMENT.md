# 62.1-08 Migration Impact Assessment

## Objective

Assess future migration impact for introducing Patient.

## Principle

Patient introduction must not require Lead migration.

## Decisions

Existing Leads: DO NOT MIGRATE. Existing Appointments: DO NOT MIGRATE. Patient records: ADDITIVE ONLY. Associations: GOVERNED ONLY.

## Guardrails

Any future migration must prove no Lead Replacement, no Appointment Replacement, no Dual Write, no Automated Patient Merge, no protected component impact, no RBAC bypass, full auditability, and rollback strategy.

## Migration Candidate Strategy

Add Patient records, add evidence capture, add candidate associations, add governed reviews, add confirmed associations, add read-only audit views.
