# 62.1-04 Authorization Matrix Design

## Objective

Define authorization controls for Patient identity actions.

## Roles

System, Authorized Reviewer, Governance Administrator, Auditor, Unauthorized User.

## Matrix Summary

| Action | System | Reviewer | Governance Admin | Auditor |
|---|---|---|---|---|
| Detect candidate | Yes | No | No | No |
| Capture evidence | Yes | Yes | Yes | No |
| Assess confidence | Yes | Yes | Yes | No |
| Start review | No | Yes | Yes | No |
| Accept/reject association | No | Yes | Yes | No |
| Correct association | No | Conditional | Yes | No |
| Restrict/unlock identity | No | No | Yes | No |
| View audit history | No | Conditional | Yes | Yes |
| Modify audit history | No | No | No | No |
| Automatic merge | No | No | No | No |

## Gates

Identity Review Gate, Sensitive Correction Gate, Audit Gate.
