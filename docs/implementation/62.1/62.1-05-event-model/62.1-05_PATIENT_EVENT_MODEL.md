# 62.1-05 Patient Event Model

## Objective

Define Patient events for identity lifecycle, governance, relationships, and auditability without Event Sourcing.

## Principles

Events describe facts, are not commands, do not become Source of Truth, and never bypass governance.

## Event Categories

Patient Identity Events, Identity Governance Events, Lead Relationship Events, Appointment Relationship Events, Administrative Governance Events.

## Standard Event Envelope

EventId, EventType, OccurredAt, EntityType, EntityId, ActorId, CorrelationId, PreviousState, NewState, Reason.

## Prohibited Events

PatientMergedAutomatically, LeadReplacedByPatient, AppointmentReplacedByPatient, IdentityApprovedWithoutReview, AuditRecordModified.
