# 62.1-02 Identity Evidence Technical Model

## Objective

Represent identity evidence, confidence, review, and audit technically without implementation.

## Model Layers

PatientId -> Identity Evidence -> Confidence Assessment -> Governance Review -> Auditable Decision.

## Evidence Object

EvidenceId, PatientId, EvidenceType, EvidenceValue, EvidenceSource, EvidenceStrength, VerificationStatus, CapturedAt, CapturedBy, CorrelationId.

## Verification Status

Unverified, Verified, Conflicting, Superseded, Restricted.

## Decision Separation

Evidence informs. Confidence evaluates. Governance decides. Audit documents.

## Prohibited Decisions

AutomaticMerge, LeadReplacement, AppointmentReplacement, SilentCorrection, UnauditedDecision.
