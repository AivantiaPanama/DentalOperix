# 62.1-07 API Boundary Analysis

## Objective

Define candidate Patient API boundaries without defining final endpoints or implementing APIs.

## Candidate Capability Groups

- Patient Identity capabilities.
- Identity Evidence capabilities.
- Identity Review capabilities.
- Relationship capabilities.
- Audit read-only capabilities.

## Prohibited API Capabilities

Merge Patients automatically, replace Lead with Patient, replace Appointment with Patient, update Lead status from Patient API, update Appointment status from Patient API, change Calendar state, send Gmail notification, modify audit records, bypass RBAC.

## Protected Component Decision

No direct impact to BookingDialog, processDentalLead, /api/leads/create, Calendar, Gmail, FloatingDentalAIChat, Home, or siteServices.ts.
