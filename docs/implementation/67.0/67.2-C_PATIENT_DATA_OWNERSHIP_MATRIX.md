# 67.2-C Patient Data Ownership Matrix

**Baseline:** DENTALOPERIX_BASELINE_66_1  
**Autoridad:** Architect Principal / Technical Reviewer / Governance Guardian  
**Fecha de actualización:** 2026-06-24  
**Idioma de comentarios:** Español

## Restricciones vigentes

- No Dual Write.
- No Lead Replacement.
- No New Lead Source of Truth.
- No Persistence Re-Architecture.
- No RBAC Bypass.
- No Automated Patient Merge.
- No modificación de componentes protegidos.

## Componentes protegidos

- BookingDialog
- processDentalLead
- /api/leads/create
- Calendar
- Gmail
- FloatingDentalAIChat
- Home
- siteServices.ts

## Objetivo

Definir la propiedad de datos entre Leads, Patients y Appointments.

## Regla fundamental

```text
1 dato
1 propietario
1 autoridad
```

Los demás dominios pueden referenciar, consultar o relacionar, pero no reclamar propiedad.

## Matriz general

| Dato                        | Source of Truth |
| --------------------------- | --------------- |
| Lead Source                 | Leads           |
| Campaign Attribution        | Leads           |
| Funnel Stage                | Leads           |
| Lead Status                 | Leads           |
| Conversion Status           | Leads           |
| Patient Identity            | Patients        |
| Patient Name                | Patients        |
| Patient Contact Information | Patients        |
| Patient DOB                 | Patients        |
| Patient Status              | Patients        |
| Appointment Date            | Appointments    |
| Appointment Time            | Appointments    |
| Appointment Status          | Appointments    |
| Provider Assignment         | Appointments    |
| Calendar References         | Appointments    |

## Referencias permitidas

- Lead puede referenciar PatientId sin ser propietario de Patient.
- Patient puede mantener referencias históricas sin administrar Lead Lifecycle.
- Appointment puede referenciar PatientId sin modificar identidad.
- Appointment puede referenciar Lead únicamente para trazabilidad histórica cuando sea necesario.

## Restricciones

- Prohibido duplicar propiedad de datos.
- Prohibido cambiar Source of Truth sin revisión formal.
- Prohibido usar Patients para reemplazar Leads.
- Prohibido usar Appointments para almacenar identidad.
- Prohibido crear ownership compartido.
