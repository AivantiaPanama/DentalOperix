# 61.4-ID-03 IDENTITY RESOLUTION STRATEGY CANDIDATES

Phase: 61.4  
Track: Identity Discovery  
Status: COMPLETE

## Objective

Evaluate possible strategies for resolving Patient identity without selecting or implementing one.

## Candidate A: Deterministic Matching

Uses strict matching rules such as name + date of birth + phone.

Advantages:

- Simple
- Auditable
- Predictable

Risks:

- High false negatives
- Brittle when data is incomplete

## Candidate B: Confidence Scoring

Attributes contribute to a confidence score.

Advantages:

- Flexible
- Better handling of incomplete data

Risks:

- More complex governance
- Requires threshold definition

## Candidate C: Human-in-the-Loop Review

System identifies candidates, but an authorized user reviews and decides.

Advantages:

- Safest approach
- Supports auditability
- Compatible with governance constraints

Risks:

- Requires operational workflow
- Requires role-based control in a future design phase

## Candidate D: Automatic Merge

System merges records automatically.

Status: PROHIBITED

Reason: Automated Patient Merge is explicitly prohibited by current governance.

## Preferred Discovery Direction

Confidence detection plus human review is the most compatible discovery direction.

The system may detect possible matches, but must not automatically merge Patients.
