# 61.4-PG-02 PATIENT AUDITABILITY MODEL

Phase: 61.4  
Track: Patient Governance Discovery  
Status: COMPLETE

## Objective

Define which Patient identity events require traceability.

## Auditable Event Categories

| Event                             | Purpose                          |
| --------------------------------- | -------------------------------- |
| PATIENT_CREATED                   | Patient identity record created. |
| PATIENT_UPDATED                   | Patient attributes updated.      |
| PATIENT_IDENTITY_CORRECTED        | Identity data corrected.         |
| PATIENT_DUPLICATE_REVIEWED        | Duplicate candidate reviewed.    |
| PATIENT_LEAD_ASSOCIATION_REVIEWED | Lead relationship reviewed.      |

## Minimum Audit Metadata

- Timestamp
- Actor
- Reason or context
- Previous value when applicable
- New value when applicable
- Related PatientId when applicable

## Prohibited Event

PATIENT_AUTO_MERGED is not permitted because Automated Patient Merge remains prohibited.

## Retention Direction

Identity-related audit records should be retained permanently or under the maximum retention policy available in a future design phase.
