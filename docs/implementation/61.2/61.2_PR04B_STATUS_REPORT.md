## Scope
PR-61.2-04B introduces controlled Lead status management from the Assistant / Front Desk Workspace.

The implementation updates only the `status` field of an existing Lead through the certified persistence port:

```text
Lead Detail
-> /api/leads/update-status
-> LeadPersistenceProvider
-> LeadPersistencePort.updateLead(id, { status })
-> RelationalLeadPersistenceAdapter
-> Supabase PostgreSQL
```

## Certified Architecture Confirmation

```text
Leads
-> LeadPersistencePort
-> LeadPersistenceProvider
-> RelationalLeadPersistenceAdapter
-> Supabase PostgreSQL
```

remains unchanged.

## Leads Source of Truth

Leads remains the single source of truth for Lead status.

This PR does not use or promote `.data/lead-operations.json` as a Lead status source.

## Functional Changes

- Added controlled status update endpoint: `/api/leads/update-status`.
- Added status selector and update action inside Lead Detail.
- Updates local Lead Queue state after a successful status update.
- Extends allowed CRM statuses with Front Desk operational states:
  - `contactado`
  - `seguimiento`
  - `no interesado`
- Keeps existing statuses:
  - `nuevo`
  - `agendada`
  - `completada`
  - `cancelada`
  - `no asistió`
