# 1. Purpose

This document records implementation-level decisions made during Iteration 61.1.

It must not be used to approve architecture changes. Any decision that requires architecture modification must be escalated to Architecture Review.

---

# 2. Decision Rules

## Rule 1

Implementation decisions must remain inside the approved 61.1 scope.

## Rule 2

Implementation decisions must not alter:

```text
Leads = Source of Truth
```

## Rule 3

Implementation decisions must not modify protected components without explicit approval.

## Rule 4

Architecture decisions must not be made by Cursor during implementation.

# 5. Required During Implementation

Cursor must add a decision entry when choosing implementation mechanisms such as:

- Authentication provider or session mechanism
- Authorization guard implementation pattern
- Route protection pattern
- Database migration approach for users and roles
- Test framework or test execution adjustments

Any decision with architecture impact must be marked:

```text
ESCALATED
```

and implementation must stop until reviewed.
