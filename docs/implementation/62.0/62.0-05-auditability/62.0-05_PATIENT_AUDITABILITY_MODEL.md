# 62.0-05 Patient Auditability Model

## Objective

Ensure Patient identity decisions are observable, reconstructable, verifiable, and explainable.

## Audit Scope

Patient identity, identity resolution, identity governance, and Patient relationships.

## Audit Categories

- Identity audit.
- Relationship audit.
- Governance audit.
- Security audit.

## Minimum Audit Envelope

Event type, timestamp, actor, entity type, entity identifier, previous state, new state, reason, correlation reference.

## Event Families

PatientCreated, PatientIdentityUpdated, IdentityCandidateDetected, IdentityEvidenceAdded, IdentityReviewStarted, IdentityReviewCompleted, LeadPatientAssociationConfirmed, AppointmentPatientAssociationCorrected, GovernanceRestrictionApplied.

## Non-Goals

Audit does not become Source of Truth, execute decisions automatically, perform merge, modify Leads, or modify Appointments.
