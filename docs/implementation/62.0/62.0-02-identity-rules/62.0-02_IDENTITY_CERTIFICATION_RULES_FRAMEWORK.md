# 62.0-02 Identity Certification Rules Framework

## Objective

Define auditable rules for evaluating whether observed records represent the same person.

## Principles

- Identity is evidence-based, not assumption-based.
- No single weak attribute certifies identity.
- Confidence and action are separate.
- Automated Patient Merge remains prohibited.

## Evidence Tiers

| Tier     | Examples                                                     |
| -------- | ------------------------------------------------------------ |
| Strong   | Verified phone, verified email, verified patient identifier. |
| Moderate | Legal name, birth date, address.                             |
| Weak     | Preferred name, nickname, city, referral context.            |

## Confidence Levels

UNKNOWN, POSSIBLE, PROBABLE, CONFIRMED, ESCALATED.

## Candidate Rules

- Name only: maximum POSSIBLE.
- Phone only: maximum PROBABLE.
- Email only: maximum PROBABLE.
- Phone + email: candidate CONFIRMED, subject to review and audit.
- Conflicting strong evidence: ESCALATED.

## Certification Formula

Evidence exists + confidence assessed + review performed + audit recorded + no automatic merge.
