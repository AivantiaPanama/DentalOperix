# 62.0-01B Patient Aggregate Boundary Definition

## Objective

Define consistency boundaries for the Patient aggregate.

## Aggregate Root

Patient.

## Inside Boundary

- Identity data.
- Contact identity.
- Address identity.
- Identity governance metadata.

## Outside Boundary

- Lead lifecycle.
- Appointment lifecycle.
- Calendar and Gmail integration behavior.
- Clinical domain.
- Financial and insurance domain.

## Candidate Invariants

1. One Patient represents one human identity.
2. Patient never contains commercial lead lifecycle state.
3. Patient never contains appointment operational state.
4. Automated merge is prohibited.
5. Relevant identity changes must be auditable.

## Architecture Compatibility

No impact to certified Lead persistence architecture or protected components.
