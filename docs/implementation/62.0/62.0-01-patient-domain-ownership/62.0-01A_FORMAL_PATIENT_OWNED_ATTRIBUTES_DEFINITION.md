# 62.0-01A Formal Patient-Owned Attributes Definition

## Objective

Define attributes owned by Patient as the identity source of truth.

## Patient-Owned Categories

| Category | Attributes |
|---|---|
| Core Identity | First name, middle name, last name, preferred name, display name, birth date, gender if used. |
| Contact Identity | Primary email, secondary emails, primary phone, secondary phones, preferred contact method. |
| Address Identity | Country, state, city, postal code, address lines. |
| Identity Governance | Identity status, confidence, review status, flags, governance metadata. |

## Excluded Attributes

Patient does not own lead source, campaign, UTM, lead status, pipeline, appointment status, appointment date/time, provider assignment, Calendar metadata, Gmail metadata, clinical facts, financial facts, or insurance coverage.

## Governance Validation

This definition preserves Leads, Patients, and Appointments as separate Sources of Truth.
