# 71.5.2 Roadmap Reordering Decision

## Program

71.5 — Controlled Development

## Increment Decision

71.5.2 is updated to be:

```text
71.5.2 — Patient Application Layer
```

The previously planned persistence-focused increment is deferred to the next controlled increment:

```text
71.5.3 — Patient Persistence
```

## Decision Type

Minor roadmap reordering.

This is a sequencing and nomenclature update only. It is not an architecture change, source-of-truth change, persistence re-architecture, or implementation authorization.

## Decision Rationale

71.5.1 established the Patient Domain Foundation and introduced the domain-level PatientPersistencePort contract. The next safest step is to add the application layer that orchestrates use cases over the port without introducing concrete infrastructure, adapters, API routes, UI dependencies, Supabase integration, or protected component changes.

This ordering preserves separation of concerns:

```text
Patient Domain Foundation
  -> Patient Application Layer
  -> Patient Persistence
  -> Patient Provider / Wiring
  -> Patient API
  -> Patient UI
  -> Integration & Validation
  -> Production Readiness
```

## 71.5.2 Scope Boundary

- Patient application services.
- Patient use cases.

- PatientPersistenceProvider.
