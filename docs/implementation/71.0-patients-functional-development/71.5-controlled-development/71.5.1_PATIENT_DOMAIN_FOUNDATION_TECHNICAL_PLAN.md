# 71.5.1 Patient Domain Foundation - Technical Plan

## Governance Context

- Baseline: DENTALOPERIX_BASELINE_69_2
- Framework: DentalOperix Governance Framework v1.0
- Maturity Level: GML-1
- Program: 71.5 Controlled Development
- Increment: 71.5.1 Patient Domain Foundation

## Scope

This increment creates a pure Patient domain foundation under `src/server/patients/domain/`.

Included:

- Patient entity factory and update function
- Patient value object normalization
- Patient enums and types
- Patient domain errors
- Patient validation rules
- PatientPersistencePort contract
- Domain-only test coverage
- Architecture guard extension

Excluded:

- API routes
- UI components
- Supabase migrations
- relational adapters
- Leads changes
- Calendar or Gmail changes
- protected component changes
- automated patient merge
