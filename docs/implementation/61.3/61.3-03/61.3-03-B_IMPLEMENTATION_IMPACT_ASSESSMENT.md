## 8. Service-Level Impact Assessment

### 8.1 PatientCreationService

Expected orchestration:

```text
Map CreatePatientCommand to CreatePatientInput
Resolve identity before create
Call PatientRepository.createPatient
Record patient_created audit event
Return PatientServiceResult<CreatePatientResult>
```

Risks:

```text
Duplicate identity handling
Ambiguous identity handling
Incorrect source mapping
Audit failure handling
```

Mitigation:

```text
Use existing domain/repository contracts.
Return deterministic application errors.
Do not write Leads.
```

### 8.2 PatientUpdateService

Expected orchestration:

```text
Map UpdatePatientCommand to UpdatePatientInput
Call PatientRepository.updatePatient
Record patient_updated audit event
Return PatientServiceResult<UpdatePatientResult>
```

Risks:

```text
Identity authority bypass
Over-broad updates
Audit failure handling
```

Mitigation:

```text
Only map fields allowed by contract.
Do not mutate identifiers outside authorized domain paths.
```

### 8.3 PatientArchiveService

Expected orchestration:

```text
Find current Patient
Update status to archived through PatientRepository.updatePatient
Record patient_updated or lifecycle audit event
Return PatientServiceResult<ArchivePatientResult>
```

Risks:

```text
Hard delete
Persistence bypass
Incomplete audit metadata
```

Mitigation:

```text
Soft archive only.
Repository contract only.
No schema or SQL changes.
```

### 8.4 PatientIdentityResolutionService

Expected orchestration:

```text
Validate at least one criterion exists
Map ResolvePatientIdentityCommand to repository identity search
Return IdentityResolutionResult
```

Risks:

```text
Empty searches
Overconfident matching
Automatic merge drift
```

Mitigation:

```text
Reject empty criteria.
Return candidates only.
No merge execution.
```

### 8.5 PatientMergePreparationService

Expected orchestration:

```text
Load source patient
Load target patient
Compare merge-relevant fields
Return MergePreparationResult with reviewRequired = true
```

Allowed optional dependency:

```text
PatientMergeContract for manual merge request preparation only if needed by implementation design.
```

Risks:

```text
Automatic merge execution
Patient deletion
Conflict masking
```

Mitigation:

```text
Manual review required.
No merge execution.
No destructive mutation.
```

### 8.6 PatientAuditService

Expected orchestration:

```text
Map RegisterPatientAuditEventCommand to PatientAuditEvent
Call PatientAuditContract.recordPatientAuditEvent
Return PatientServiceResult<RegisterPatientAuditEventResult>
```

Risks:

```text
Audit persistence ownership drift
Unsupported audit event type
```

Mitigation:

```text
Use PatientAuditContract only.
No direct audit table writes.
```

## 9. Risk Assessment

| Risk | Severity | Likelihood | Mitigation |
|---|---:|---:|---|
| Business rules move into services | High | Medium | Keep domain authoritative; services only map/orchestrate |
| Repository bypass | High | Low | Import scan for Supabase/SQL/direct adapter usage |
| Lead boundary violation | Critical | Low | Protected import scan and file diff review |
| UI coupling | High | Low | Import scan for React/routes/components |
| Automatic merge introduced | Critical | Low | Enforce `reviewRequired = true`; no destructive writes |
| Audit failure ambiguity | Medium | Medium | Deterministic `PatientServiceResult` error mapping |
| Existing `patient-service.ts` conflict | Medium | Medium | Preserve existing service; introduce new application layer alongside it |
