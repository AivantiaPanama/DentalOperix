## 7. Service Authorization Matrix

| Service | Authorized Responsibility | Forbidden Responsibility |
|---|---|---|
| PatientCreationService | Coordinate identity pre-check, domain/repository create, audit registration | Own duplicate rules, write Leads, call Supabase directly |
| PatientUpdateService | Coordinate repository update and audit registration | Bypass domain validation, overwrite identity authority |
| PatientArchiveService | Coordinate archive state transition through repository update and audit | Hard delete records, mutate schema, alter Leads |
| PatientIdentityResolutionService | Convert application command into identity search / domain resolution call | Execute merge, persist identity changes |
| PatientMergePreparationService | Prepare manual-review merge data and/or request manual merge contract | Execute automatic merge |
| PatientAuditService | Register patient audit events through PatientAuditContract | Own persistence, write audit tables directly |

## 8. Required Error Handling

Services must return `PatientServiceResult<T>` and map failures into `PatientApplicationError[]`.

Required behavior:

```text
No unhandled domain/repository/audit exception should cross the application service boundary unless explicitly justified.
Application errors must be deterministic and typed by code/message.
```

Recommended error codes:

```text
PATIENT_VALIDATION_ERROR
PATIENT_NOT_FOUND
PATIENT_DUPLICATE_IDENTITY
PATIENT_AMBIGUOUS_IDENTITY
PATIENT_AUDIT_FAILED
PATIENT_IDENTITY_RESOLUTION_FAILED
PATIENT_MERGE_PREPARATION_FAILED
PATIENT_APPLICATION_UNEXPECTED_ERROR
```
