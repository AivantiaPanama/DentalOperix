## 8. Implementation Guardrails

The implementation batch must follow these guardrails:

```text
Create service files only under src/server/patients/application/services/
Use contracts from src/server/patients/application/contracts/
Use shared result/error/context types from src/server/patients/application/types/
Return PatientServiceResult<T> from application services
Map errors into PatientApplicationError[]
Keep business rules in domain layer
Use PatientRepository contract only for persistence operations
Use PatientAuditContract only for audit registration
Do not import from UI, routes, Leads, Supabase, Calendar or Gmail
Do not execute automatic merge
```
