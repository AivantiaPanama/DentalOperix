# 1. Purpose

Define the formal contract specification for Patient Application Services.

These contracts are:

```text
Application DTOs
Application Commands
Application Results
Application Errors
Application Context
```

These contracts are not:

```text
Domain Entities
Persistence Models
Database Schemas
UI Models
```

---

# 2. Shared Types

## PatientServiceContext

Purpose:

Provide execution metadata for application services.

Fields:

| Field         | Type   | Required |
| ------------- | ------ | -------- |
| actorId       | string | YES      |
| actorType     | string | YES      |
| correlationId | string | YES      |
| timestamp     | Date   | YES      |

Constraints:

```text
No business rules
No persistence references
```

---

## PatientServiceResult<T>

Purpose:

Standard service response wrapper.

Fields:

| Field   | Type                      |
| ------- | ------------------------- |
| success | boolean                   |
| data    | T                         |
| errors  | PatientApplicationError[] |

---

## PatientApplicationError

Purpose:

Standard application error model.

Fields:

| Field   | Type   |
| ------- | ------ |
| code    | string |
| message | string |
| details | string |

# 3. Create Patient

## CreatePatientCommand

Purpose:

Request Patient creation.

Fields:

| Field       | Type                  | Required |
| ----------- | --------------------- | -------- | --- |
| firstName   | string                | YES      |
| lastName    | string                | YES      |
| dateOfBirth | Date                  | null     | NO  |
| phones      | string[]              | NO       |
| emails      | string[]              | NO       |
| source      | string                | YES      |
| context     | PatientServiceContext | YES      |

Rules:

```text
No patientId allowed
No persistence identifiers allowed
```

| createdAt       | Date    |
| auditRegistered | boolean |
