## 10. Risk Assessment

| Risk | Severity | Mitigation |
|---|---|---|
| Service responsibility drift | Medium | Keep business rules in Domain. |
| Repository bypass | High | Services depend only on repository contracts. |
| Lead boundary violation | Critical | No Lead persistence imports allowed. |
| UI coupling | High | No UI imports allowed. |
| Duplicate service ownership with patient-service.ts | Medium | Preserve existing PatientService and introduce application layer without breaking existing behavior. |
