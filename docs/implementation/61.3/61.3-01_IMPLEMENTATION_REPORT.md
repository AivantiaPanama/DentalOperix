## 4. Certified Architecture Preservation

The certified Leads architecture remains unchanged:

```text

```

No protected component was modified.

No Dual Write, Lead Replacement, new Leads Source of Truth, Persistence Re-Architecture, Analytics Write Back, RBAC Bypass, automated merge, insurance benefits logic, clinical records, treatment plan, or UI implementation was introduced.

## 5. Domain Boundary

Patient is implemented as a permanent identity aggregate.

Included in aggregate:

```text
Patient
PatientPhone
PatientEmail
PatientAddress
PatientIdentifier
```

External references only:

```text
linkedLeadId
linkedAppointmentId
```

These references do not replace Lead or Appointment source-of-truth ownership.
