## 7. Rollback Consideration

The migration uses `CREATE TABLE IF NOT EXISTS`, `CREATE INDEX IF NOT EXISTS`, and guarded enum creation blocks. This is safe for forward application but does not include an automatic destructive rollback.

Recommended rollback policy:

```text
No automatic rollback script for production.
Manual rollback only under explicit DBA / architecture approval.
```

Reason:

```text
Patient identity data, once created, may become business-critical and must not be dropped automatically.
```
