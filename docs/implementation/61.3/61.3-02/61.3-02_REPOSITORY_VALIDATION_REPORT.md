## 7. Identity Resolution Repository Support

File:

```text
src/server/patients/patient-identity-resolution.ts
```

Repository method used:

```ts
searchPatientsByIdentity(search)
```

Supported search inputs:

```text
normalizedName
email
phone
identifierType
identifierValue
excludePatientId
```

### Validation

| Identity Signal | Repository Support | Result |
| --- | --- | --- |
| Normalized name | Yes | PASS |
| Normalized email | Yes | PASS |
| Normalized phone | Yes | PASS |
| Identifier type/value | Yes | PASS |
| Exclusion by patient ID | Intended | PASS WITH OBSERVATION |

### Observation

`searchPatientsByIdentity` builds an exclusion clause for `excludePatientId`. The final SQL includes the exclusion correctly in the `WHERE` clause, although the local `clauses` array is also appended after `identityWhere` is already computed. This does not appear to break the final query because the final SQL explicitly composes:

```text
identityWhere AND p.id <> $N
```

**Result:** PASS WITH MINOR IMPLEMENTATION STYLE OBSERVATION.

### H-61.3-02-003 — Repository Validation Cannot Execute Runtime Tests in This Audit Environment

**Status:** OPEN / NON-BLOCKING
**Severity:** Low
**Area:** Local validation environment

Attempted validation using `npm ci --ignore-scripts` could not proceed because `package.json` and `package-lock.json` are not synchronized. `npm ci` reported a missing `lru-cache@11.5.1` lockfile entry.

**Recommendation:** Refresh lockfile or use the project's canonical package manager before executing the full runtime suite.
