## 4. Alignment Decisions

- 61.3 initial implementation should be Patient Identity Foundation, not UI-first Patient Workspace.
- Patient identity is permanent and immutable at the identity level.
- Contact data is mutable and normalized.
- Duplicate handling must be controlled and auditable.
- Insurance, benefits, treatment and billing remain bounded future domains.
- Domain Discovery Gate is now an adopted governance pattern for new domains.
