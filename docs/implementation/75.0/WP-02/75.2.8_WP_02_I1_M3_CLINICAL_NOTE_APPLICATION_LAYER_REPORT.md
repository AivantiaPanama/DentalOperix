## 1. Purpose

## 4. Design Determination

The application layer is organized around use cases rather than a single procedural service.

This preserves one responsibility per intention:

- registering a clinical note;
- completing a clinical note;
- reopening a clinical note;
- amending a clinical note;
- archiving a clinical note.

The application layer delegates clinical rules to `ClinicalNoteDomainService` and depends only on `ClinicalNoteRepositoryPort` for persistence abstraction.
