## 1. Purpose

This micro-increment connects the Clinical Notes application layer to the certified Clinical Records persistence architecture through a controlled relational adapter.

The objective is to introduce persistence infrastructure for Clinical Notes without changing the domain model, application use cases, Sources of Truth, protected components, or the certified persistence strategy.

## 3. Persistence Design

The new table is `clinical_notes`.

It stores clinical narrative records owned exclusively by Clinical Records.

Key fields include:

- `id`
- `clinical_record_id`
- `patient_id`
- `appointment_id`
- `title`
- `narrative`
- `status`
- audit timestamps
- healthcare professional audit references
- `source`

The table includes indexes for:

- `clinical_record_id`
- `patient_id, created_at DESC`
- `status`
