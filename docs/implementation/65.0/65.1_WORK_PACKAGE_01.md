# 65.1 WORK PACKAGE 01

## Patient Domain Foundation

Version: 1.0  
Status: APPROVED  
Program Phase: 65.0 Implementation Execution  
Work Package ID: 65.1-WP01

---

# 1. Purpose

Define the first executable work package under 65.0 Implementation Execution.

This work package establishes the Patient Domain Foundation within the certified DentalOperix architecture and governance constraints.

---

# 2. Objective

Create the foundation required to support patient identity management while preserving all certified Sources of Truth and architectural boundaries.

---

# 3. Scope

Included:

- Patient Domain Foundation
- Patient Identity Model
- Patient Persistence Structures
- Patient Repository Layer
- Patient Auditability Support
- RBAC Compliance Validation

Excluded:

- Appointment redesign
- Lead lifecycle changes
- Lead replacement
- Automated patient merge
- Protected component modifications

---

# 4. Architectural Constraints

Certified architecture remains unchanged:

```text
Leads
→ LeadPersistencePort
→ LeadPersistenceProvider
→ RelationalLeadPersistenceAdapter
→ Supabase PostgreSQL
```

No re-architecture is authorized.

---

# 5. Source of Truth Constraints

Leads remains Source of Truth for:

- acquisition
- marketing
- lead lifecycle

Patients remains Source of Truth for:

- person identity

Appointments remains Source of Truth for:

- scheduled operational events

---

# 6. Protected Components

No modification authorized for:

- BookingDialog
- processDentalLead
- /api/leads/create
- Calendar
- Gmail
- FloatingDentalAIChat
- Home
- siteServices.ts
