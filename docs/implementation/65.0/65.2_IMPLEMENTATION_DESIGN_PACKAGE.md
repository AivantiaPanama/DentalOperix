# 65.2 IMPLEMENTATION DESIGN PACKAGE

## DentalOperix

Version: 1.0  
Status: APPROVED  
Program Phase: 65.0 Implementation Execution  
Work Package: 65.1-WP01 Patient Domain Foundation

---

# 1. Purpose

Define the executable implementation design for the Patient Domain Foundation authorized under WP01.

---

# 2. Scope

Included:

- Patient Domain Foundation
- Patient Identity Model
- Patient Repository Layer
- Patient Persistence Structures
- Patient Auditability Support
- RBAC Validation

Excluded:

- Lead Lifecycle Changes
- Appointment Redesign
- Protected Component Changes
- Architecture Redesign
- Automated Patient Merge

---

# 3. Domain Boundary Design

Patients Domain responsibilities:

- Person identity management
- Patient identity persistence
- Patient auditability
- Patient record ownership

Not responsible for:

- Lead acquisition lifecycle
- Marketing lifecycle
- Appointment scheduling authority

---

# 4. Source of Truth Design

| Domain | Authority |
|---|---|
| Leads | acquisition, marketing, lead lifecycle |
| Patients | person identity |
| Appointments | scheduled operational events |

These responsibilities are immutable within WP01.

---

# 5. Logical Component Design

Primary components:

- Patient Entity
- Patient Repository
- Patient Validation Layer
- Patient Audit Layer

Supporting components:

- RBAC Validation
- Governance Validation

---

# 6. Persistence Design

WP01 patient persistence must remain compatible with the certified persistence architecture.

No alternate persistence mechanism is authorized. No persistence re-architecture is authorized.

---

# 7. Repository Design

Required repository responsibilities:

- Create Patient
- Read Patient
- Update Patient
- Search Patient

Repository requirements:

- Auditability
- RBAC enforcement
- Governance compliance

---

# 8. Validation Design

Required validations:

- Architectural Validation
- Source of Truth Validation
- RBAC Validation
- Regression Validation
- Governance Validation

All validations must PASS.

---

# 9. Evidence Requirements

- E1 Architecture Compliance Evidence
- E2 Source of Truth Compliance Evidence
- E3 RBAC Compliance Evidence
- E4 Governance Compliance Evidence
- E5 Regression Evidence
