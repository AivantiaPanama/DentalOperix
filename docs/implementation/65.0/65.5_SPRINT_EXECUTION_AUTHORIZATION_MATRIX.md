# 1. Purpose

Define the governance authorization model controlling execution of every WP01 sprint.

---

# 2. Authorization Principles

No sprint may begin unless:

- Previous sprint completed
- Evidence collected
- Governance review completed
- Sprint authorization granted

---

---

# 4. Escalation Rules

Immediate escalation required for:

- Architecture violation
- Source of Truth violation
- RBAC violation
- Protected component impact
- Governance non-compliance

---

# 5. Protected Component Restrictions

No sprint is authorized to modify:

- BookingDialog
- processDentalLead
- /api/leads/create
- Calendar
- Gmail
- FloatingDentalAIChat
- Home
- siteServices.ts
