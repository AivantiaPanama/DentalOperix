# 65.0 Execution Governance Checklist

## DentalOperix

Version: 1.0  
Status: ACTIVE  
Applies To: Every 65.x work package

---

## 1. Purpose

Provide the mandatory checklist that must be completed before any 65.x work package can proceed to implementation activity.

This checklist does not authorize code generation by itself.

---

## 2. Mandatory Pre-Implementation Review

Every work package must include:

- [ ] Architectural analysis
- [ ] Affected dependencies
- [ ] Risk assessment
- [ ] Technical impact assessment
- [ ] Compatibility with 57.x
- [ ] Compatibility with 61.x
- [ ] Compatibility with 62.x
- [ ] Compatibility with 63.0
- [ ] Compatibility with 64.0
- [ ] Governance determination
- [ ] Explicit approval before code generation

---

## 3. Architecture Guardrails

Certified architecture:

```text
Leads
-> LeadPersistencePort
-> LeadPersistenceProvider
-> RelationalLeadPersistenceAdapter
-> Supabase PostgreSQL
```

Required result:

```text
ARCHITECTURE COMPLIANCE: PASS
```

---

## 4. Source of Truth Guardrails

- Leads = acquisition, marketing and lead lifecycle
- Patients = person identity
- Appointments = scheduled operational events

Required result:

```text
SOURCE OF TRUTH COMPLIANCE: PASS
```

---

## 5. Protected Component Guardrails

Protected components:

- BookingDialog
- processDentalLead
- /api/leads/create
- Calendar
- Gmail
- FloatingDentalAIChat
- Home
- siteServices.ts

Default result:

```text
PROTECTED COMPONENT IMPACT: NONE
```

Any impact requires explicit architecture and governance approval.

---

## 6. Prohibited Activities

The work package must not introduce:

- Dual Write
- Lead Replacement
- New Lead Source of Truth
- Persistence Re-Architecture
- RBAC Bypass
- Automated Patient Merge

Required result:

```text
PROHIBITED ACTIVITY: NONE
```

---

