# 57.7 Cutover Execution Planning

## Status

STATUS: ACTIVE

## Executive Approval

The Executive Authorization Hold has been formally released for planning.

This release authorizes execution planning only. It does not by itself authorize runtime cutover until the final Go/No-Go package is completed and accepted.

## Architectural Guardrails

- Leads remains the Source of Truth.
- Google Sheet remains the current physical persistence until final cutover execution.
- Relational Database is the future physical persistence.
- Dual Write is not allowed.
- Multiple Sources of Truth are not allowed.
- Projection Engine is not allowed.
- Product Migration is not allowed.

## Required Environment Preparation

A relational database connection does not currently exist in the provided environment. The repository now includes:

- `.env.relational.example`
- `scripts/validate-relational-env.mjs`
- `npm run validate:relational-env`

## Required User Steps

1. Provision the approved relational database.
2. Copy `.env.relational.example` to `.env.relational.local`.
3. Fill the approved database connection string in `DATABASE_URL` or `POSTGRES_URL`.
4. Keep `LEADS_PERSISTENCE_MODE=google-sheet` for configuration validation.
5. Run `npm run validate:relational-env -- .env.relational.local`.
6. Only during an approved cutover window may `LEADS_PERSISTENCE_MODE=relational-db` and `RELATIONAL_CUTOVER_APPROVED=true` be used together.
