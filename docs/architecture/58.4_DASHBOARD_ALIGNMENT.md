## Purpose

58.4 aligns administrative dashboard consumers to the 58.3 Revenue Intelligence integration boundary and begins reducing direct dashboard dependency on scattered legacy metric calculations.

The phase uses a compatibility adapter so existing UI components can consume a dashboard-compatible view while the authoritative KPI source becomes `RevenueSnapshotV1`.

## Compatibility Adapter

58.4 adds:

```ts
fetchRevenueDashboardMetrics(period?: string | null)
```

Location:

```text
src/lib/api/revenue-dashboard-metrics.ts
```

The adapter performs:

```text
RevenueSnapshotV1
  -> CrmDashboardMetrics-compatible view
  -> Existing dashboard widgets
```

This avoids broad UI rewrites while making `RevenueSnapshotV1` the primary dashboard KPI source.

## Compatibility Notes

Some legacy dashboard fields do not exist in `RevenueSnapshotV1` v1, including lead score distribution, urgency breakdown, and period-over-period comparison. The compatibility adapter supplies neutral values for those fields to preserve current component contracts.

Those fields should be evaluated in future phases before removal or formal inclusion in a later Revenue Intelligence contract version.
