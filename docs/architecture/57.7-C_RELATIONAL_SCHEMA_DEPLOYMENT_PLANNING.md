## Risks

### RISK-57.7-C-001: Accidental Cutover Interpretation

Creating tables could be misinterpreted as enabling relational persistence.

Mitigation:

- Deployment script refuses to run unless `LEADS_PERSISTENCE_MODE=google-sheet`.
- Deployment script refuses to run if `RELATIONAL_CUTOVER_APPROVED=true`.
- Deployment requires explicit schema-only flags.

### RISK-57.7-C-002: Historical Status Compatibility

Production snapshot showed both `new` and `nuevo` values.

Mitigation:

- 57.7-C SQL permits both `new` and `nuevo`.
- No data normalization is performed in this phase.

### RISK-57.7-C-003: Unauthorized Writes

Schema deployment writes database DDL only.

Mitigation:

- No lead records are inserted.
- No production traffic is routed.
- No adapter activation is performed.

## Implementation Plan

### Step 1: Validate environment

```bash
npm run validate:relational-env -- .env.relational.local
```

### Step 2: Validate connectivity

```bash
npm run validate:relational-connectivity -- .env.relational.local
```

### Step 3: Enable schema-only deployment flags

Only when ready to create tables:

```env
LEADS_PERSISTENCE_MODE=google-sheet
RELATIONAL_CUTOVER_APPROVED=false
RELATIONAL_CONNECTIVITY_TEST_ENABLED=true
RELATIONAL_SCHEMA_DEPLOYMENT_APPROVED=true
RELATIONAL_SCHEMA_DEPLOYMENT_ENABLED=true
```

### Step 4: Deploy schema

```bash
npm run deploy:relational-schema -- .env.relational.local
```

### Step 5: Validate schema

```bash
npm run validate:relational-schema -- .env.relational.local
```
