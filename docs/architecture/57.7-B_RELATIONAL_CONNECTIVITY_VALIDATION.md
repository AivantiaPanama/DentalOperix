# 57.7-B Relational Connectivity Validation

## Status

READY FOR LOCAL EXECUTION

## Governance

This validation is a connectivity-only test. It does not authorize cutover and does not change the active persistence backend.

Required controls:

```text
LEADS_PERSISTENCE_MODE=google-sheet
RELATIONAL_CUTOVER_APPROVED=false
RELATIONAL_CONNECTIVITY_TEST_ENABLED=true
```

## Purpose

Validate that DentalOperix can establish a read-only PostgreSQL connection to the approved Supabase database endpoint and execute:

```sql
SELECT 1;
```

## Scope

In scope:

- DATABASE_URL / POSTGRES_URL parsing
- Supabase PostgreSQL network connectivity
- TLS connection
- SELECT 1 execution

Out of scope:

- Cutover
- Schema migration
- Writes
- Dual write
- Provider switch to relational-db
- Runtime persistence activation

## Command

```bash
npm install
npm run validate:relational-env -- .env.relational.local
npm run validate:relational-connectivity -- .env.relational.local
```

If `pg` is missing, run:

```bash
npm install pg
```

## Expected Result

```text
Result: PASS
```

## Failure Conditions

The validation must fail if:

- DATABASE_URL / POSTGRES_URL is missing.
- LEADS_PERSISTENCE_MODE is not `google-sheet`.
- RELATIONAL_CUTOVER_APPROVED is not `false`.
- RELATIONAL_CONNECTIVITY_TEST_ENABLED is not `true`.
- PostgreSQL connection fails.
- SELECT 1 fails.

## Governance Decision

A PASS result means connectivity is verified only.

It does not mean:

- relational persistence is active,
- cutover is approved,
- data migration is approved,
- production execution is approved.
