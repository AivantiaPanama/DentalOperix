## Incident

A user reported that after selecting date and time in the booking dialog, clicking **Confirmar reserva** produced no visible action. The expected behavior is:

```text
click Confirmar reserva
  -> visible submitting state
  -> call createDentalAppointment
  -> persist lead through certified provider path
  -> show visible success confirmation or visible error
```

## Architectural Analysis

`BookingDialog` is a protected component. It was modified only after explicit approval for this hotfix.

The hotfix is UI/action-guard scoped. It does not alter:

- `processDentalLead`
- `/api/leads/create`
- Calendar integration behavior
- Gmail integration behavior
- persistence adapters
- runtime source of truth

## Implemented Changes

1. Added explicit submit telemetry for the booking confirmation action:

```text
booking_submit_clicked
booking_submit_started
booking_submit_failed
```

2. Added client-side action guard:

```text
ignore duplicate submit while saving
show visible validation error when date/time is incomplete
clear stale server error before a valid submit
```

3. Added button-level safeguards:

```text
type="button"
disabled while saving
aria-busy while saving
visible Confirmando state with spinner
```

4. Kept footer visible during submission so the user sees immediate action feedback while the content area shows the loading view.

5. Added focused UI tests for:

```text
selected appointment -> click Confirmar reserva -> createDentalAppointment called -> success confirmation visible
missing date/time -> click Confirmar reserva -> visible validation error -> no server call
```
