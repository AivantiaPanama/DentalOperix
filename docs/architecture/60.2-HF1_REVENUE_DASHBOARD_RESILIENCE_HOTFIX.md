## Trigger

Runtime testing reported a dashboard failure caused by:

```text
GET /api/analytics/revenue?period=all 500 (Internal Server Error)
```

The failure propagated through:

```text
fetchRevenueIntelligence
  -> fetchRevenueDashboardMetrics
  -> admin dashboard
```

## Architectural Decision

Revenue Intelligence remains the primary dashboard metrics source.

A controlled fallback was added so the dashboard can continue rendering from the legacy CRM metrics API when the Revenue Intelligence endpoint is temporarily unavailable.

```text
Dashboard
  -> fetchRevenueDashboardMetrics
      -> Revenue Intelligence API (primary)
      -> CRM Metrics API (fallback only on failure)
```
