# 60.3-HF2 Dashboard Empty State Guard Hotfix

## Status

IMPLEMENTED

## Context

Manual validation after 60.3-HF1 confirmed that Google Sheets and Revenue Intelligence were operating correctly:

```text
/api/analytics/revenue?period=all
success: true
degraded: false
source: google-sheets
snapshot.totals.leads: 18
```

However, the first navigation to the Admin Dashboard could still show the empty CRM message:

```text
Aún no existen registros en el CRM.
```

A browser-side fetch performed while the empty message was visible confirmed that the backend already had real data. Therefore, the issue was classified as a dashboard UI empty-state guard problem, not a Google Sheets, Revenue Intelligence, or persistence issue.

## Scope

Updated:

```text
src/routes/admin/dashboard.tsx
src/routes/admin/-dashboard.test.tsx
```

## Change Summary

The dashboard now derives the empty CRM visual state from both:

```text
metrics.emptyCRM === true
```

and:

```text
metrics.totals.leads === 0
```

The empty CRM message is only shown after loading is complete and only when no leads are present.

## Implementation Pattern

```text
shouldShowEmptyCRM = !loading && metrics.emptyCRM === true && metrics.totals.leads === 0
```

This prevents a stale or transitional `emptyCRM` flag from showing an empty-state banner when the current snapshot contains real leads.
