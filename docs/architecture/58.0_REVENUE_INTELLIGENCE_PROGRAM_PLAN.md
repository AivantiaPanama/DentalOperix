# 58.0 Revenue Intelligence Program Plan

## Status

PROGRAM: 58.x Revenue Intelligence
STATUS: INITIATED
GOVERNANCE: ACTIVE
ARCHITECTURE CHANGE: NO
SOURCE OF TRUTH CHANGE: NO
DUAL WRITE: NO

## Governing Baseline

This program is governed by:

- `docs/architecture/57.9_DOCUMENTATION_CONSOLIDATION_AND_PROGRAM_CLOSURE.md`

The 57.x persistence transition remains closed and certified. Revenue Intelligence must not reopen persistence governance, change the certified source of truth, or introduce an alternate master data source.

Certified baseline:

```text
Leads
  -> LeadPersistencePort
  -> LeadPersistenceProvider
  -> RelationalLeadPersistenceAdapter
  -> Supabase PostgreSQL
```

Certified source of truth:

```text
Leads = Source of Truth
```

## Program Objective

Revenue Intelligence converts existing lead and CRM operational data into governed read-only business metrics.

The program does not create master records, does not persist alternate lead state, and does not replace the certified lead persistence architecture.

## Architectural Position

```text
Leads Source of Truth
  -> Read / Analytics Calculations
  -> Revenue Intelligence Contract
  -> Dashboards / Reports / Insights
```

Revenue Intelligence is a derived analytics layer.

## Protected Components

The program must not modify:

- `BookingDialog`
- `processDentalLead`
- `/api/leads/create`
- Calendar integration
- Gmail integration
- `FloatingDentalAIChat`
- `Home`
- `siteServices.ts`

## Initial Scope

58.2 introduces `RevenueSnapshotV1`, a typed read-only contract that centralizes existing CRM and commercial KPI formulas into one governed output shape.

## Out of Scope

- New persistence tables
- Dual write
- Product migration
- Lead replacement
- New source of truth
- Mutation of lead creation or booking flows
- Modification of certified 57.x architecture

## Governance Notes

Any future endpoint, dashboard integration, or report must consume Revenue Intelligence as a derived read model and must explicitly preserve the 57.9 certified baseline.
