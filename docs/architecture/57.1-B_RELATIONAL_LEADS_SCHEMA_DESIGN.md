# 57.1-B Relational Leads Schema Design

## Status

DESIGNED / NOT ACTIVE / NOT EXECUTED

## Governance Decision

This artifact defines the target relational persistence schema for the existing Leads domain.

It does not activate relational persistence.
It does not approve dual write.
It does not change the Source of Truth.
It does not modify operational flows.

## Source of Truth

Logical Source of Truth:

- Leads

Current physical persistence:

- Google Sheet

Future physical persistence:

- Relational Database

The future relational database is a replacement for the physical persistence layer only. It is not a new domain and not a new Source of Truth.

## Scope

57.1-B introduces:

- a target `leads` relational table design
- a migration audit table design
- schema metadata in TypeScript for governance and future adapter implementation
- an offline SQL reference script

57.1-B does not introduce:

- database connection code
- runtime reads from relational storage
- runtime writes to relational storage
- production migration execution
- cutover
- rollback execution

## Target Table

Primary table:

- `leads`

Purpose:

- operational physical persistence for the existing Leads Source of Truth after future cutover approval

Important rule:

- this table must not become active while Google Sheet remains the active persistence mechanism unless a formally approved cutover plan says so

## Technical Metadata

The table includes minimal technical fields:

- `physical_source`
- `source_record_hash`
- `schema_version`
- `inserted_at`
- `updated_at`

These fields support reconciliation, certification and migration traceability. They are not new business concepts.

## Migration Audit Table

Table:

- `lead_persistence_migration_audit`

Purpose:

- track future backfill and reconciliation batches
- support certification of Google Sheet to relational database migration
- provide cutover evidence

This table is not a Source of Truth and must not be consumed by operational flows.

## Cutover Boundary

57.1-B does not approve cutover.

Cutover remains blocked until the following are approved:

1. Backfill plan
2. Reconciliation plan
3. Rollback plan
4. Runtime adapter activation plan
5. Google Sheet archival or read-only policy
6. Post-cutover certification

## Completion Criteria

57.1-B is complete when:

- target relational schema is documented
- offline SQL reference exists
- TypeScript schema metadata exists
- governance test exists
- no runtime flow is changed
- Google Sheet remains the active persistence mechanism
