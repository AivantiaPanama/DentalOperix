# 60.2-HF4 Analytics Endpoint Resilience Hotfix

## Status

IMPLEMENTED

## Governance baseline

Reference baseline:

- `docs/architecture/57.9_DOCUMENTATION_CONSOLIDATION_AND_PROGRAM_CLOSURE.md`
- `docs/architecture/60.0_CLINICAL_INTELLIGENCE_PROGRAM_PLAN.md`

This hotfix does not reopen persistence transition 57.x and does not modify the certified source-of-truth decision.

## Problem

The dashboard could receive HTTP 500 responses from read-only analytics endpoints when the Google Sheets lead source was unavailable:

- `/api/analytics/revenue?period=all`
- `/api/analytics/executive?period=all`

This created noisy client-side errors and partial dashboard degradation even though the affected endpoints are analytics/read-only.

## Scope

Updated read-only analytics endpoints:

- `src/routes/api/analytics/revenue.ts`
- `src/routes/api/analytics/executive.ts`

Updated API response typing:

- `src/lib/api/revenue-intelligence.ts`
- `src/lib/api/executive-analytics.ts`

Updated tests:

- `src/routes/api/analytics/revenue.test.ts`
- `src/routes/api/analytics/executive.test.ts`

## Design

When Google Sheets is unavailable, the endpoints now return HTTP 200 with a safe degraded payload instead of HTTP 500.

Revenue degraded response shape:

```json
{
  "success": true,
  "degraded": true,
  "source": "empty-fallback",
  "period": "all",
  "warning": "Revenue Intelligence source unavailable; returned an empty read-only snapshot.",
  "snapshot": {}
}
```

Executive degraded response shape:

```json
{
  "success": true,
  "degraded": true,
  "source": "empty-fallback",
  "period": "all",
  "warning": "Executive Analytics source unavailable; returned an empty read-only snapshot.",
  "executive": {}
}
```

The snapshots are produced through the same contract factories used by normal operation, but with an empty lead list.

## Validation checklist

- Revenue endpoint returns normal payload when Google Sheets is available.
- Revenue endpoint returns degraded payload instead of 500 when Google Sheets fails.
- Executive endpoint returns normal payload when Google Sheets is available.
- Executive endpoint returns degraded payload instead of 500 when Google Sheets fails.
- Existing production access restriction remains unchanged.
- RBAC checks remain unchanged.

## Result

60.2-HF4 improves dashboard stability by keeping analytics endpoints contract-compatible during upstream read failures.
