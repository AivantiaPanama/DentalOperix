## 3. Risks

### Risk: over-normalization

A broad encoding conversion could alter legitimate user-entered text.

Mitigation: 60.3 uses a conservative replacement table for observed mojibake sequences only. It does not rewrite arbitrary strings and does not persist normalized values back to Google Sheets.

### Risk: analytics fragmentation

Without normalization, analytics can treat these as different services:

```text
Diseño de Sonrisa
DiseÃ±o de Sonrisa
```

Mitigation: CRM service grouping and estimated pipeline valuation now normalize service names before grouping and lookup.
