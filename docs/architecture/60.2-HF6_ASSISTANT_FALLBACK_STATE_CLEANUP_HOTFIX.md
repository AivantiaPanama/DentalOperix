# 60.2-HF6 Assistant Fallback State Cleanup Hotfix

## Status

IMPLEMENTED

## Reference Architecture

This hotfix remains compatible with the certified 57.9 baseline:

```text
Leads
-> LeadPersistencePort
-> LeadPersistenceProvider
-> RelationalLeadPersistenceAdapter
-> Supabase PostgreSQL
```

The certified governance decision remains unchanged:

```text
Leads = Source of Truth
```

## Problem

Manual validation confirmed that Google Sheets reads were working correctly through:

```text
GET /api/leads/list
```

The endpoint returned real lead data without `fallback: true`, but the Assistant UI could still display the fallback banner:

```text
Datos demostrativos
No se pudo leer Google Sheets, usando demo.
```

This indicated a UI state cleanup issue rather than a Google Sheets connectivity failure.

## Scope

Updated file:

```text
src/components/assistant/AssistantDashboard.tsx
```

No changes were made to:

```text
BookingDialog
processDentalLead
/api/leads/create
Calendar
Gmail
FloatingDentalAIChat
Home
siteServices.ts
```

## Implementation

The Assistant data loader now clears any previous fallback state at the beginning of each refresh:

```text
fallback: false
message: null
```

When `/api/leads/list` returns successfully, the UI now treats fallback as active only when the response explicitly includes:

```text
fallback === true
```

The message is also retained only for explicit fallback responses.

## Expected Result

When `/api/leads/list` returns real Google Sheets data:

```json
{ "leads": [...] }
```

Assistant no longer shows the demo-data banner.

When `/api/leads/list` explicitly returns fallback data:

```json
{ "leads": [...], "fallback": true, "message": "..." }
```

Assistant still shows the controlled fallback banner.

## Technical Impact

```text
Persistence: 0
Source of Truth: 0
Google Sheets API: 0
API contract: 0
Assistant UI: visual state cleanup only
```

## Risk

LOW

The change only affects local UI state handling in AssistantDashboard and does not modify backend reads, writes, persistence, or certified lead ingestion.
