# 60.2-HF5 Admin Home Data Resilience Hotfix

## Executive Summary

60.2-HF5 hardens legacy Admin Home data endpoints that still returned HTTP 500 when Google Sheets-backed reads failed.

Affected endpoints:

```text
/api/followups/history
```

## Problem Statement

Admin Home continued to invoke legacy endpoints from `AdminHomeDashboard`:

```text
/api/crm/metrics?period=thisMonth
/api/followups/history
```

Both endpoints depended on Google Sheets read paths and returned HTTP 500 when the external read failed.

This created noisy browser console errors and incomplete Admin Home data loading even though the UI could safely operate with empty degraded datasets.

## Architectural Decision

Convert both endpoints from fail-fast behavior to fail-soft behavior.

Before:

```text
Google Sheets read failure
  -> HTTP 500
  -> UI error/noisy console
```

After:

```text
Google Sheets read failure
  -> HTTP 200
  -> degraded: true
  -> source: empty-fallback
  -> safe empty payload
```

### Followups History Endpoint

File:

```text
src/routes/api/followups/history.ts
```

Behavior changed:

```text
readAutomationRunRecords failure
  -> records: []
  -> degraded: true
  -> source: empty-fallback
  -> status: 200
```

```text
Unauthorized -> 401
```
