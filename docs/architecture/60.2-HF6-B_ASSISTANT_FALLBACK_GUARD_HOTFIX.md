# 60.2-HF6-B Assistant Fallback Guard Hotfix

## Status

IMPLEMENTED.

## Purpose

Prevent the Assistant dashboard from showing the demonstrative-data warning when `/api/leads/list` returns real leads from Google Sheets.

## Root Cause

`AssistantDashboard` displayed the fallback banner based only on `payload.fallback === true`.

During validation, `/api/leads/list` was confirmed to return real Google Sheets data:

```text
{ leads: Array(18) }
```

without `fallback: true`. However, the UI still displayed the fallback alert. To make the UI state resilient, the fallback banner must be suppressed whenever the payload contains real leads.

## Change

Updated:

```text
src/components/assistant/AssistantDashboard.tsx
```

The component now derives fallback state using both conditions:

```text
payload.fallback === true
AND
leads.length === 0
```

This ensures that real Google Sheets data always takes precedence over stale or residual fallback state.

## Validation Criteria

- `/api/leads/list` returns real leads.
- `Datos demostrativos` is not displayed when real leads exist.
- The fallback banner is still allowed only when the API explicitly returns fallback data and no real leads are present.
