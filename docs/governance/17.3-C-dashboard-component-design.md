# 17.3-C — Dashboard Component Design

## Resultado

APPROVED FOR FUTURE UI IMPLEMENTATION GATE.

Esta fase define contratos de componentes y composición de paneles para una futura UI. No implementa UI visual, rutas, APIs nuevas, persistencia, lógica de dominio ni lógica de fallback.

## Componentes candidatos

### Executive Dashboard

- PlatformHealthWidget → PlatformHealthMetric
- AggregateHealthSummaryWidget → AggregateHealthMetric
- DomainHealthOverviewWidget → DomainHealthMetric
- ExecutiveGovernanceSummaryWidget → GovernanceHealthMetric

### Operational Dashboard

- DomainHealthMatrixWidget → DomainHealthMetric
- OperationalStatusWidget → AggregateHealthMetric + DomainHealthMetric
- DomainTrendSummaryWidget → DomainHealthMetric

### Governance Dashboard

- GovernanceHealthWidget → GovernanceHealthMetric
- AdrComplianceWidget → GovernanceHealthMetric
- IsolationStatusWidget → GovernanceHealthMetric
- ReadModelFreezeStatusWidget → GovernanceHealthMetric

## Gobernanza preservada

- Metric-only components
- No raw telemetry exposure
- No aggregate access
- No adapter access
- No read source access
- No direct read model access
- No lead write path access
- Aggregate Isolation preserved
- Domain Ownership preserved
- Leads remains Source of Truth
- READ MODEL PLATFORM v2 remains CLOSED/FROZEN/GOVERNANCE BASELINED

## Dependencias permitidas

- ExecutiveDashboardApiContracts
- ExecutiveDashboardContracts
- PlatformHealthMetric
- DomainHealthMetric
- AggregateHealthMetric
- GovernanceHealthMetric

## Dependencias prohibidas

- Aggregates
- Adapters
- Raw telemetry
- Read sources
- ReadModelSourceProvider
- Direct read models
- Lead write paths
- Persistence
- Domain logic
- Fallback logic
