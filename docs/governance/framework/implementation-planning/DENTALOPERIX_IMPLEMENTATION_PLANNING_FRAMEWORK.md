# Implementation Planning Framework

**Generated:** 2026-06-25  
**Framework:** DentalOperix Engineering & Governance Framework  
**Status:** Reusable framework component.

## Purpose

Reusable structure for planning implementation after architecture validation.

## Required Sections

- Scope.
- Files/areas expected to change.
- Dependencies affected.
- Risks.
- Technical impact.
- Rollout and rollback posture.
- Evidence expected from user-run validation.
- Explicit authorization gate.

## Restriction

Planning does not authorize code. Code requires explicit user authorization.
