# 61.0 AI Delivery Process

Status: ACTIVE
Owner: AI Delivery Governance
Last updated: 2026-06-20

## Purpose

Define the delivery process for AI-generated work so the project can move quickly without producing unreviewed or incompatible changes.

## Delivery Classes

### Class A - Documentation Only

Examples:

- User stories.
- Acceptance criteria.
- Roadmap updates.
- Prompt libraries.

Review: ChatGPT + Product Owner.

### Class B - UI Mockup Only

Examples:

- v0 dashboard mockups.
- Figma layouts.

Review: UX + architecture boundary check.

### Class C - Code Candidate

Examples:

- Cursor-generated CRUD.
- Tests.
- Forms.

Review: Full implementation checklist.

### Class D - Protected Component Candidate

Examples:

- BookingDialog.
- processDentalLead.
- Gmail.
- Calendar.

Review: Explicit approval required before generation and before integration.

## Required Deliverables Per Code Candidate

- Code changes.
- Tests.
- Documentation update.
- Validation summary.
- Rollback note.
- ZIP or branch reference.

## Acceptance Gate

No candidate is accepted unless:

```text
Build passes.
Tests pass or limitation is documented.
Architecture is compatible.
Documentation is updated.
No protected component was modified without approval.
```
