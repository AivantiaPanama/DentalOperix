# 61.0 AI Driven Product Development Strategy

Status: ACTIVE
Owner: AI Delivery Governance
Last updated: 2026-06-20

## Purpose

Define how DentalOperix uses multiple AI tools to accelerate product delivery without losing architectural control.

## Operating Principle

```text
AI accelerates execution.
DentalOperix governance controls architecture.
```

No AI-generated output is accepted into the product without review against certified architecture and implementation checklist.

## AI Team Structure

| AI / Tool           | Role                                      | Primary Responsibility                                                 | Not Allowed To Own                    |
| ------------------- | ----------------------------------------- | ---------------------------------------------------------------------- | ------------------------------------- |
| ChatGPT             | Architect Principal / Governance Guardian | Architecture, planning, review, integration strategy                   | Blind implementation without approval |
| Claude              | Documentation Factory                     | User stories, business rules, acceptance criteria, permission matrices | Architecture decisions                |
| Cursor              | Implementation Factory                    | Code, CRUD, forms, tests, focused refactors                            | Source-of-truth decisions             |
| v0                  | Dashboard/UI Factory                      | Dashboard layouts and React UI mockups                                 | Business logic or data persistence    |
| Figma AI / UX tools | UX Factory                                | Wireframes, flows, design system                                       | Runtime architecture                  |

## Required Package Before Any AI Work

Every AI must receive:

1. `docs/ai-context/DENTALOPERIX_AI_CONTEXT.md`
2. Relevant iteration file from `docs/iterations/`
3. Relevant module specification.
4. Allowed and prohibited file list.
5. Acceptance criteria.
6. Architecture constraints.

## Review Gate

AI output is classified as one of:

- ACCEPT
- ACCEPT WITH CHANGES
- REWORK
- REJECT

Only ACCEPT or ACCEPT WITH CHANGES may proceed to integration.

## Prohibited AI Behavior

- Redefining Leads source of truth.
- Creating new persistence paths without approval.
- Writing from Analytics to operational data.
- Modifying protected components without explicit approval.
- Introducing security shortcuts or RBAC bypass.
- Creating demo/fallback data that masks real data.

## Expected Benefit

This model allows one human operator to coordinate a virtual product team while maintaining architectural consistency.
