# 61.0 AI Workflow Control Center

Status: ACTIVE
Owner: AI Delivery Governance
Last updated: 2026-06-20

## Purpose

This is the official workflow for assigning DentalOperix tasks to ChatGPT, Claude, Cursor, v0, Figma AI, or other tools.

## Standard Workflow

```text
1. Identify product need
2. Confirm governing documents
3. Define iteration
4. Define AI assignment
5. Create task package
6. Execute AI task
7. Review deliverable
8. Validate architecture
9. Run tests / manual validation
10. Update documentation
11. Package deliverable
12. Certify or reject
```

## Task Package Minimum Fields

```text
Task ID:
```

## Review Checklist

Before accepting an AI deliverable:

- Does it preserve Leads = Source of Truth?
- Does it avoid Dual Write?
- Does it avoid RBAC bypass?
- Does it respect protected components?
- Does it update documentation?
- Does it include tests or validation criteria?
- Does it match the assigned scope?

## Escalation Rule

If an AI proposes an architecture change, stop the task and return to ChatGPT governance review.
